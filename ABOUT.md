# アプリ企画書：マテンダー (Matender) ## 1. コンセプト 「予定を親、メモを子」とした階層型カレン

ダー・メモアプリ。 カレンダーの日付を単なる枠組みとするのではなく、その中に配置される「予定（オ

ブジェクト）」を主役とし、一つの予定に対して複数の付箋（子要素）やファイルを紐付けて管理する。

## 2. ターゲット・利用シーン - 会議やイベントごとに複数のアジェンダ、メモ、ファイルをまとめて管理

したいユーザー。 - Google Keepのような気軽さと、カレンダーの時系列管理を両立させたいユー

ザー。 - JSON等を用いたデータの一括投入・管理を行いたいパワーユーザー。 ## 3. 主要機能

### 3.1 カレンダー・ベースのUI - 標準画面はカレンダービュー。 - 日付を選択すると、その日に紐

付く「予定（親）」をリスト表示。 - 予定をタップすることで、その内部の「子メモ（付箋）」や「ファイル」へ

アクセス。 ### 3.2 階層型データ管理 - **親要素（Parent）：** 予定、タスク、イベント。 - 日付・時刻

の設定が可能（時間指定なしも可）。 - ドラッグ＆ドロップで日付変更が可能（子要素も自動追従）。 -

**子要素（Child）：** メモ、付箋、ファイル。 - 一つの親に対して無制限に追加可能。 - 親の中での並

び替え、チェックリスト化。 ### 3.3 データ同期・入出力 - **ハイブリッド同期:** ローカル（SQLite等）

での高速動作と、クラウド（Supabase等）でのマルチデバイス同期。 - **JSON流し込み機能:** 外部

で作成したJSONファイルをインポートし、親子構造を維持したまま一括で予定・メモを作成。 ## 4.

技術スタック（推奨） - **フレームワーク:** React Native + Expo (Android対応) - **カレンダーライブ

ラリ:** react-native-calendars - **データベース:** SQLite (Local) + Supabase (Cloud) - **UIコン

ポーネント:** Tailwind CSS (NativeWind) または Lucide Icons ## 5. データ構造案 (JSON) ```json

{ "event_id": "uuid", "title": "プロジェクトMTG", "date": "2026-04-17", "time": "10:00", "children": [ {

"file", "name": "資料.pdf", "url": "..." }, { "id": "m3", "type": "memo", "content": "次回のスケジュール

調整", "is_done": false } ] } ``` ## 6. 今後の展望 - 音声入力による予定・子メモの自動生成。 - ファ
